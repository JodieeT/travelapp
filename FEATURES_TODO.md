## 一、用户端预定流程（移动端）

### 1. 酒店查询页（首页）［5 分］

- [ ] **顶部 Banner**
  - [ ] 展示酒店广告（来自后端 Banner 数据）
  - [ ] 点击某条 Banner 跳转到该酒店的详情页
- [ ] **核心查询区域**
  - [ ] **当前地点**：支持定位（如浏览器 Geolocation API 或选城市）
  - [ ] **关键字搜索**：输入框，用于搜索酒店名/地址等
  - [ ] **酒店入住日期选择**：开发日历组件，选择入住日期
  - [ ] **筛选条件**：酒店星级、价格等（可自行设计交互）
  - [ ] **快捷标签**：酒店属性标签（如亲子、豪华、免费停车场等，可与后端 tags 对应）
- [ ] **查询按钮**：点击后携带上述条件跳转到酒店列表页

### 2. 酒店列表页［15 分］

- [ ] **顶部核心条件筛选头**
  - [ ] 城市
  - [ ] 入住 / 离店日期
  - [ ] 入住间夜（可由入住、离店日期计算或单独展示）
  - [ ] 搜索设置（与首页关键字一致或延续）
- [ ] **详细筛选区域**：可自行设计（如星级、价格区间、标签等）
- [ ] **酒店列表**
  - [ ] 列表项展示：酒店名、评分（如有）、地址、价格等（维度可自行优化）
  - [ ] **上滑自动加载**：滚动到底部时加载下一页（分页/无限滚动）

### 3. 酒店详情页［15 分］

- [ ] **顶部导航头**：显示酒店名称 + 返回列表页
- [ ] **大图 Banner**：当前酒店图片，支持左右滑动/滚动查看
- [ ] **酒店基础信息**：酒店名、星级、设施、地址（点评与地图可不实现）
- [ ] **日历 + 入住间夜 Banner**：展示入住/离店及间夜信息
- [ ] **房型价格列表**：按商户端数据价格从低到高排序展示

## 二、管理酒店信息系统（PC 站点）

### 4. 用户登录页 / 注册［5 分］

- [ ] **注册界面**
  - [x] 用户名、密码
  - [x] **角色选择**：注册时可选「商户」或「管理员」
  - [ ] _密码确认_
  - [ ] _人机验证_
  - [ ] _管理员注册权限_
- [ ] **登录界面**
  - [x] 用户名、密码
  - [x] **不选角色**：登录后由后端/账号自动判断角色并跳转
  - [ ] _更有用的报错信息_
  - [ ] _找回密码_
- [ ] **角色与权限**
  - [x] 商户：上传/编辑酒店信息
  - [x] 管理员：审核上传及发布酒店信息
  - [x] 不能访问与角色不符的页面

### 5. 酒店信息录入 / 编辑 / 修改［10 分］（角色：商户）

- [x] **酒店信息录入**：新建酒店时可填写全部维度
- [x] **酒店信息编辑 / 修改**：对已有酒店（草稿或已驳回）进行编辑
- [x] **保存功能**：提交后持久化到后端
- [ ] **保存后实时更新到端侧**：保存后列表/详情立即反映新数据（含价格等）

**数据维度（必须）**

- [x] 酒店名（中文）
- [x] 酒店名（英文）
- [x] 酒店地址
- [x] 酒店星级
- [x] 酒店房型（多种房型）
- [x] 酒店价格（各房型对应价格）
- [x] 酒店开业时间

**数据维度（可选，可提升体验）**

- [ ] 酒店附近热门景点、交通、商场
- [ ] 酒店价格打折/优惠（如节日 8 折、机酒套餐减多少元等）

### 6. 酒店信息审核 / 发布 / 下线列表页［10 分］（角色：管理员）

- [x] **列表展示**：展示待审核/已发布/已下线等酒店
- [x] **审核状态**：通过 / 不通过 / 审核中
- [x] **不通过时显示原因**：驳回理由展示在列表中或详情中
- [x] **发布**：审核通过即发布（状态为已发布）
- [x] **下线**：对已发布酒店执行下线（非物理删除）
- [x] **恢复**：对已下线酒店可恢复为已发布
